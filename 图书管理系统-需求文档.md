## 1. 项目概述

### 1.1 项目背景
随着图书馆藏书量的不断增加和读者服务需求的多样化，传统的人工管理方式已无法满足现代图书馆的管理需求。本项目旨在开发一套基于Web的图书管理系统，实现图书馆业务的数字化、自动化管理。

### 1.2 项目目标
开发一个基于B/S（浏览器/服务器）架构的图书管理系统，实现：
- 图书信息的数字化管理
- 借阅流程的自动化处理
- 读者服务的在线化
- 管理决策的数据化支持

### 1.3 目标用户
| 用户角色 | 描述 | 主要需求 |
|---------|------|---------|
| 普通读者 | 图书馆注册会员、市民 | 在线检索图书、查看借阅记录、续借图书 |
| 系统管理员 | 图书馆工作人员 | 图书管理、用户管理、借阅管理、数据统计 |

### 1.4 项目价值
- 提高图书管理效率，减少人工操作
- 减少人为错误，提升数据准确性
- 提高图书流通率和利用率
- 增强信息透明度，提升读者体验
- 为管理决策提供数据支持

## 2. 系统功能需求

### 2.1 功能架构图

```
图书管理系统
├── 前台读者门户
│   ├── 用户认证模块
│   ├── 图书检索与浏览模块
│   └── 个人中心模块
└── 后台管理面板
    ├── 仪表盘模块
    ├── 图书管理模块
    ├── 分类管理模块
    ├── 用户管理模块
    ├── 借阅管理模块
    └── 系统设置模块
```

### 2.2 前台读者门户功能详述

#### 2.2.1 用户认证模块

**功能点1: 用户注册**
- 必填项：用户名、密码、确认密码、真实姓名、手机号/邮箱
- 验证要求：
  - 用户名唯一性验证
  - 密码强度验证（至少8位，包含字母和数字）
  - 手机号/邮箱格式验证
  - 发送验证码进行手机号/邮箱验证
- 注册成功后自动跳转到登录页面

**功能点2: 用户登录**
- 支持用户名/手机号/邮箱登录
- 密码输入框支持显示/隐藏切换
- "记住我"选项（保持7天登录状态）
- 登录失败提示：账户不存在/密码错误
- 连续输错5次密码锁定账户15分钟

**功能点3: 密码找回**
- 通过注册手机号/邮箱找回密码
- 发送验证码验证身份
- 设置新密码并确认
- 密码重置成功后强制重新登录

**功能点4: 用户注销**
- 用户可申请注销账户
- 注销前需验证密码
- 提示：注销后数据不可恢复，借阅中的图书需先归还
- 有未归还图书的账户不允许注销

#### 2.2.2 图书检索与浏览模块

**功能点1: 首页搜索**
- 搜索框：支持关键词模糊搜索
  - 可搜索字段：书名、作者、出版社、ISBN
  - 实时搜索建议（输入关键词时显示匹配的图书）
- 搜索结果页：
  - 按相关度排序
  - 支持按"可借状态"、"分类"筛选
  - 支持按"书名"、"作者"、"出版日期"排序

**功能点2: 高级搜索**
- 组合搜索条件：
  - 书名（模糊匹配）
  - 作者（模糊匹配）
  - ISBN（精确匹配）
  - 出版社
  - 分类（支持多选）
  - 出版年份范围
  - 可借状态（全部/可借/已借完）

**功能点3: 分类导航**
- 展示方式：左侧树形分类导航 + 右侧图书列表
- 分类结构：支持二级分类
  - 一级分类示例：文学、历史、科技、经济等
  - 二级分类示例：文学 > 中国文学、外国文学等
- 点击分类自动加载该分类下的图书

**功能点4: 图书列表页**
- 显示内容：
  - 图书封面缩略图（150x200px）
  - 书名（粗体显示）
  - 作者
  - 出版社
  - 可借状态标签：可借（绿色）/已借完（灰色）
- 分页显示：每页20本
- 列表视图/网格视图切换

**功能点5: 图书详情页**
- 基本信息：
  - 图书封面大图
  - 书名、副标题
  - 作者、译者
  - 出版社、出版日期
  - ISBN、定价
  - 图书分类
- 馆藏信息：
  - 馆藏总数
  - 可借数量
  - 当前借出数量
- 内容简介
- 目录（如有）
- 相关推荐（同分类或同作者的其他图书）

#### 2.2.3 个人中心模块

**功能点1: 我的借阅**

**当前借阅**
- 表格字段：
  - 图书封面缩略图
  - 书名
  - 借阅日期
  - 应还日期（逾期用红色标注）
  - 剩余天数（逾期显示"已逾期X天"）
  - 是否已续借
  - 操作按钮：续借（仅可续借一次，已续借的显示为灰色不可点击）
- 续借规则：
  - 每本书仅可续借一次
  - 续借后延长借阅期限（根据系统设置的续借天数）
  - 已逾期的图书不可续借

**借阅历史**
- 表格字段：
  - 书名
  - 借阅日期
  - 归还日期
  - 是否逾期
  - 逾期费用（如有）
- 支持按时间范围筛选
- 支持导出个人借阅记录

**功能点2: 个人信息**
- 显示信息：
  - 头像（支持上传修改）
  - 用户名（不可修改）
  - 真实姓名
  - 手机号
  - 邮箱
  - 注册时间
  - 借阅统计：累计借阅次数、当前借阅数
- 可编辑项：
  - 头像
  - 真实姓名
  - 手机号（需验证码验证）
  - 邮箱（需验证码验证）

**功能点3: 修改密码**
- 输入项：
  - 当前密码
  - 新密码
  - 确认新密码
- 验证规则：
  - 当前密码正确性验证
  - 新密码强度验证
  - 新密码与当前密码不能相同
- 修改成功后强制重新登录

### 2.3 后台管理面板功能详述

#### 2.3.1 管理员登录
- 独立的管理员登录入口（/admin/login）
- 登录验证：管理员账号 + 密码 + 验证码
- 会话管理：30分钟无操作自动退出
- 权限校验：所有管理功能访问前验证管理员身份

#### 2.3.2 仪表盘模块

**数据卡片**
- 图书总数
  - 显示馆藏图书总数
  - 环比增长/减少百分比
- 用户总数
  - 显示注册用户总数
  - 环比增长百分比
- 当日借阅量
  - 显示今日借阅图书数量
  - 同比昨日增长/减少
- 逾期未还数量
  - 显示当前逾期未还图书数量
  - 逾期图书列表链接

**数据图表**
- 近30天借阅趋势图（折线图）
  - 横坐标：日期
  - 纵坐标：借阅数量
  - 可切换查看：借书量、还书量
- 热门借阅图书TOP 10（柱状图）
  - 显示借阅次数最多的10本图书
  - 可切换时间范围：本周、本月、本年
- 图书分类分布（饼图）
  - 显示各分类图书数量占比

**快捷操作**
- 快速借书
- 快速还书
- 添加新图书
- 查看待处理逾期

#### 2.3.3 图书管理模块

**功能点1: 图书列表**
- 列表字段：
  - 图书ID
  - 封面缩略图
  - 书名
  - 作者
  - ISBN
  - 分类
  - 馆藏总数
  - 可借数量
  - 操作按钮：编辑、删除
- 搜索功能：
  - 支持按书名、作者、ISBN搜索
  - 支持按分类筛选
- 批量操作：
  - 批量删除（需二次确认）
  - 批量导出为Excel
- 分页：每页显示50条

**功能点2: 添加图书**
- 表单字段：
  - ISBN（支持扫码枪输入，自动识别后填充书名、作者等信息）
  - 书名（必填）
  - 副标题
  - 作者（必填）
  - 译者
  - 出版社（必填）
  - 出版日期（必填，日期选择器）
  - 图书分类（必填，下拉选择）
  - 定价（必填，数字输入）
  - 入库数量（必填，数字输入）
  - 图书封面（图片上传）
  - 内容简介（富文本编辑器）
  - 目录（富文本编辑器）
- 验证规则：
  - ISBN唯一性验证（已存在则提示）
  - 必填项验证
  - 价格和数量为正数
- 提交成功后跳转到图书列表页

**功能点3: 编辑图书**
- 加载已有信息到表单
- 可修改除ISBN外的所有字段
- 支持调整馆藏数量（增加/减少）
- 修改数量时需填写原因（如：损坏、丢失、补充入库等）
- 保存修改记录到操作日志

**功能点4: 删除图书**
- 删除前二次确认弹窗
- 验证：该书有未归还的借阅记录则不允许删除
- 删除成功后记录到操作日志

**功能点5: 批量导入图书**
- 提供Excel模板下载
- 上传Excel文件进行批量导入
- 导入前数据验证：
  - 必填项完整性检查
  - ISBN格式验证
  - 分类是否存在验证
- 导入结果反馈：
  - 成功导入X条
  - 失败Y条，显示失败原因

**功能点6: 批量导出图书**
- 支持导出所有图书或筛选后的图书
- 导出格式：Excel
- 导出字段：ISBN、书名、作者、出版社、出版日期、分类、定价、馆藏总数、可借数量

#### 2.3.4 分类管理模块

**功能点1: 分类列表**
- 树形结构展示分类
- 显示字段：
  - 分类名称
  - 分类级别（一级/二级）
  - 图书数量
  - 创建时间
  - 操作按钮：编辑、删除、添加子分类

**功能点2: 添加分类**
- 输入项：
  - 分类名称（必填）
  - 上级分类（一级分类则选择"无"）
  - 排序序号（数字，用于排序）
- 验证：分类名称在同级中不重复

**功能点3: 编辑分类**
- 可修改分类名称和排序序号
- 可调整上级分类（需验证不形成循环）

**功能点4: 删除分类**
- 删除前验证：该分类下有图书则不允许删除
- 删除父分类时，如有子分类需先删除子分类
- 删除前二次确认

#### 2.3.5 用户管理模块

**功能点1: 用户列表**
- 列表字段：
  - 用户ID
  - 头像
  - 用户名
  - 真实姓名
  - 手机号
  - 邮箱
  - 注册时间
  - 账户状态（正常/禁用）
  - 当前借阅数
  - 操作按钮：查看详情、禁用/启用、重置密码
- 搜索功能：
  - 支持按姓名、手机号、邮箱搜索
- 筛选功能：
  - 按账户状态筛选（全部/正常/禁用）
- 分页：每页显示30条

**功能点2: 用户详情**
- 基本信息展示
- 借阅统计：
  - 累计借阅次数
  - 当前借阅图书列表
  - 借阅历史记录
  - 逾期次数
  - 累计逾期费用
- 操作日志：登录记录、密码修改记录等

**功能点3: 添加用户**
- 表单字段：
  - 用户名（必填，唯一）
  - 密码（必填）
  - 真实姓名（必填）
  - 手机号（必填）
  - 邮箱
- 用于管理员手动添加用户（如为不熟悉系统的读者开通账户）

**功能点4: 禁用/启用用户**
- 禁用用户：
  - 禁用后用户无法登录
  - 需填写禁用原因
  - 已借图书不受影响，但不能续借
- 启用用户：
  - 恢复正常登录和借阅权限

**功能点5: 重置密码**
- 管理员可为用户重置密码为默认密码
- 重置后用户首次登录强制修改密码

#### 2.3.6 借阅管理模块

**功能点1: 借书操作**
- 输入方式：
  - 用户信息：用户ID/姓名/手机号
  - 图书信息：ISBN/书名（支持扫码枪）
- 自动校验：
  - 用户账户状态（是否禁用）
  - 用户是否达到最大借书数量
  - 用户是否有逾期未还图书
  - 图书是否可借（是否有库存）
- 校验通过后：
  - 显示确认信息（用户姓名、图书名称、应还日期）
  - 点击确认完成借阅
  - 扣减图书可借数量
  - 生成借阅记录

**功能点2: 还书操作**
- 输入方式：
  - 图书ISBN或扫描条形码
- 自动识别借阅记录
- 计算是否逾期：
  - 未逾期：提示正常归还
  - 已逾期：显示逾期天数和应缴费用
- 确认还书后：
  - 更新借阅记录（归还日期、是否逾期、逾期费用）
  - 增加图书可借数量
  - 如有逾期费用，记录到用户账户

**功能点3: 借阅记录查询**
- 筛选条件：
  - 用户姓名/ID
  - 图书名称/ISBN
  - 借阅日期范围
  - 归还日期范围
  - 借阅状态（全部/借阅中/已归还/已逾期）
- 列表字段：
  - 借阅ID
  - 用户姓名
  - 图书名称
  - 借阅日期
  - 应还日期
  - 实际归还日期
  - 是否续借
  - 是否逾期
  - 逾期天数
  - 逾期费用
- 导出功能：导出为Excel

**功能点4: 逾期管理**
- 逾期图书列表：
  - 显示所有当前逾期未还的图书
  - 列表字段：用户姓名、手机号、图书名称、应还日期、逾期天数、应缴费用
  - 操作：催还通知（发送短信/邮件提醒）
- 逾期统计：
  - 按用户统计逾期次数
  - 按图书统计逾期频率

#### 2.3.7 系统设置模块

**功能点1: 借阅规则设置**
- 配置项：
  - 最大借书数量（默认5本）
  - 借阅期限（默认30天）
  - 续借期限（默认15天）
  - 每本书可续借次数（默认1次）
  - 逾期每日罚款金额（默认0.5元/天）
- 修改后立即生效（对新借阅生效，已有借阅不受影响）

**功能点2: 系统信息**
- 显示系统版本号
- 显示数据库信息
- 显示服务器状态

**功能点3: 操作日志**
- 记录所有管理员操作
- 日志字段：
  - 操作时间
  - 操作人
  - 操作类型（添加/修改/删除图书、用户管理、借还书等）
  - 操作内容
  - IP地址
- 支持按时间范围和操作类型筛选

## 4. 业务流程

### 4.1 读者注册流程
```
开始 → 填写注册信息 → 验证手机号/邮箱 → 提交注册 → 系统验证 → 注册成功 → 跳转登录页
                                                    ↓
                                                验证失败 → 提示错误信息 → 返回修改
```

### 4.2 图书借阅流程（读者端）
```
开始 → 搜索图书 → 查看详情 → 确认可借 → 到馆借阅（线下） → 管理员操作借书 → 借阅成功
                                ↓
                            已借完 → 等待归还或查看其他图书
```

### 4.3 图书借阅流程（管理员端）
```
开始 → 输入用户信息 → 输入图书ISBN → 系统校验 → 校验通过 → 确认借阅 → 生成借阅记录 → 完成
                                        ↓
                                    校验失败 → 显示失败原因 → 结束
```

### 4.4 图书归还流程
```
开始 → 扫描图书ISBN → 识别借阅记录 → 计算是否逾期 → 确认还书 → 更新记录 → 完成
                                        ↓
                                    已逾期 → 显示逾期费用 → 确认缴费 → 完成
```

### 4.5 续借流程
```
开始 → 查看当前借阅 → 点击续借 → 验证续借资格 → 更新应还日期 → 续借成功
                                    ↓
                                不符合条件 → 提示原因 → 结束
```

### 4.6 图书入库流程
```
开始 → 扫描ISBN → 检查是否已存在 → 不存在 → 填写图书信息 → 提交 → 保存成功
                                    ↓
                                已存在 → 提示增加馆藏数量 → 更新数量 → 保存
```

## 6. 非功能性需求

### 6.1 性能要求
- **响应时间**: 
  - 页面加载时间 < 3秒
  - 搜索响应时间 < 2秒
  - 借还书操作响应时间 < 1秒
- **并发能力**: 
  - 支持至少100个用户同时在线
  - 支持至少20个管理员同时操作
- **数据库性能**:
  - 图书表支持至少10万条记录
  - 借阅记录表支持至少100万条记录

### 6.2 安全性要求
- **数据安全**:
  - 密码使用bcrypt算法加密存储
  - 敏感信息（手机号、邮箱）加密存储
  - 数据库定期备份（每日凌晨自动备份）
- **应用安全**:
  - 防止SQL注入攻击（使用参数化查询）
  - 防止XSS攻击（输入输出过滤）
  - 防止CSRF攻击（使用Token验证）
  - 文件上传安全验证（限制文件类型和大小）
- **会话安全**:
  - 使用HTTPS协议传输数据
  - Session超时时间设置（普通用户2小时，管理员30分钟）
  - 同一账户异地登录强制下线
- **权限控制**:
  - 前端路由守卫：未登录用户重定向到登录页
  - 后端接口权限验证：每个请求验证用户身份和权限
  - 管理员操作需二次验证（如删除、禁用用户等敏感操作）

### 6.3 可用性要求
- **易用性**:
  - 界面简洁直观，符合用户操作习惯
  - 重要操作提供操作引导
  - 提供清晰的操作反馈（成功、失败、错误提示）
  - 表单验证提供友好的错误提示
- **兼容性**:
  - 支持主流浏览器：Chrome、Firefox、Safari、Edge（最近两个版本）
  - 响应式设计，支持PC端和移动端访问
  - 最小屏幕分辨率：1366x768
- **可访问性**:
  - 支持键盘导航
  - 合理的颜色对比度
  - 重要信息提供文字说明

### 6.4 可靠性要求
- **系统稳定性**:
  - 系统可用性达到99%以上
  - 无单点故障，关键服务支持高可用
- **数据可靠性**:
  - 数据一致性保证（使用事务处理）
  - 定期数据备份（全量备份 + 增量备份）
  - 备份数据异地存储
- **错误处理**:
  - 友好的错误提示页面（404、500等）
  - 系统异常自动记录日志
  - 关键错误邮件/短信通知管理员

### 6.5 可维护性要求
- **代码质量**:
  - 代码结构清晰，模块化设计
  - 遵循编码规范
  - 关键代码添加注释
  - 函数和方法职责单一
- **文档完整性**:
  - API接口文档（使用Swagger）

  - 访问日志：记录用户访问行为

## 7. 技术实现要求

### 7.1 技术栈
- **前端**: 
  - 框架：Vue.js 3.x
  - UI组件库：Element Plus
  - 状态管理：Pinia
  - 路由：Vue Router
  - HTTP客户端：Axios
  - 图表：ECharts
- **后端**:
  - 语言：Golang
  - 框架：Gin
  - ORM：GORM
  - 认证：JWT
  - 配置管理：Viper
- **数据库**:
  - 主数据库：MySQL 8.0+ / PostgreSQL 13+
  - 缓存：Redis（可选，用于Session和热点数据）
- **其他**:
  - API设计：RESTful风格
  - 接口文档：Swagger
  - 版本控制：Git

### 7.2 开发规范
- **前端规范**:
  - 使用ESLint进行代码检查
  - 使用Prettier进行代码格式化
  - 组件命名使用PascalCase
  - 文件名使用kebab-case
- **后端规范**:
  - 遵循Go官方编码规范
  - 使用golint/golangci-lint进行代码检查
  - API接口遵循RESTful设计原则
  - 错误使用统一的错误码体系

### 7.3 API设计规范
- **请求格式**:
  - 使用JSON格式传输数据
  - GET请求参数使用URL参数
  - POST/PUT/DELETE请求参数使用请求体
- **响应格式**:
```json
{
  "code": 200,
  "message": "成功",
  "data": {}
}
```
- **HTTP状态码**:
  - 200: 成功
  - 400: 请求参数错误
  - 401: 未授权
  - 403: 无权限
  - 404: 资源不存在
  - 500: 服务器错误
- **业务错误码**（自定义code字段）:
  - 1000: 用户不存在
  - 1001: 密码错误
  - 1002: 用户已被禁用
  - 2000: 图书不存在
  - 2001: 图书库存不足
  - 3000: 达到最大借书数量
  - 3001: 存在逾期图书不能借阅
  - 3002: 已达到续借次数上限
