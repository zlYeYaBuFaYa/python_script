### 1.3 技术选型

| 领域 | 技术方案 | 理由 |
|------|---------|------|
| UI 框架 | SwiftUI | iOS 16+ 原生支持，声明式 UI 开发效率高 |
| 数据持久化 | CoreData + NSPersistentContainer | Apple 原生 ORM，支持轻量级迁移，iCloud 预留能力 |
| 音频引擎 | AVFoundation (AVPlayer) | 原生支持所有P0格式，内存映射播放大文件 |
| 元数据解析 | ID3TagEditor (MIT) | 轻量级 ID3 解析库，支持 MP3/M4A |
| HTTP 服务器 | GCDWebServer (BSD-3) | 成熟稳定的 iOS 内嵌 HTTP 服务器 |
| 图片缓存 | 自研 LRU (基于 NSCache + FileManager) | 需求简单，无需引入 SDWebImage 等重量级库 |
| 依赖注入 | Swift 原生协议 + @Environment | 避免引入第三方 DI 框架，简洁可控 |
| Hash 计算 | CryptoKit (SHA256) | 系统原生，用于文件去重 |
| 二维码生成 | CoreImage (CIQRCodeGenerator) | 系统原生，WiFi 传输地址二维码 |
| 主题管理 | ThemeManager (ObservableObject) + @Environment | 可扩展多主题架构，v1.0 内置默认主题 |

### 1.4 第三方依赖清单

| 库名称 | 版本 | 许可证 | 用途 | 引入阶段 |
|--------|------|--------|------|---------|
| ID3TagEditor | 4.x | MIT | ID3 元数据解析 | P0 |
| GCDWebServer | 3.x | BSD-3 | WiFi 传输 HTTP 服务器 | P1 |

> 设计原则：最小化第三方依赖。能用系统 API 实现的不引入第三方库。

### 2.2 CoreData Entity 详细定义

#### 2.2.1 SongEntity（歌曲）

| 属性 | 类型 | 可选 | 默认值 | 索引 | 说明 |
|------|------|------|--------|------|------|
| `id` | UUID | N | auto | PK | 唯一标识 |
| `title` | String | N | "" | Y (复合) | 歌曲名称 |
| `artist` | String | N | "未知艺术家" | Y (复合) | 艺术家 |
| `album` | String | N | "未知专辑" | Y (复合) | 专辑 |
| `duration` | Double | N | 0 | N | 时长（秒） |
| `fileHash` | String | N | "" | Y (唯一约束组) | SHA256 文件哈希 |
| `fileRelativePath` | String | N | "" | N | 相对 Documents/Music/ 的路径 |
| `coverImagePath` | String | Y | nil | N | 封面图相对 Caches/ 路径 |
| `lyricsPath` | String | Y | nil | N | 歌词文件相对路径 |
| `lyricsSource` | Int16 | Y | nil | N | 0=embedded, 1=file, 2=manual |
| `format` | String | N | "" | Y | 音频格式 (mp3/m4a/aac/alac/wav) |
| `sampleRate` | Int32 | Y | nil | N | 采样率 (Hz) |
| `bitRate` | Int32 | Y | nil | N | 比特率 (kbps) |
| `channels` | Int16 | Y | nil | N | 声道数 |
| `encoder` | String | Y | nil | N | 编码器 |
| `trackNumber` | Int16 | Y | nil | N | 音轨编号 |
| `year` | Int16 | Y | nil | N | 发行年份 |
| `genre` | String | Y | nil | N | 流派 |
| `importDate` | Date | N | now | Y | 导入时间 |
| `fileSize` | Int64 | N | 0 | N | 文件大小（字节） |
| `playCount` | Int32 | N | 0 | N | 播放次数 |
| `isFavorite` | Bool | N | false | Y | 是否收藏 |
| `favoriteDate` | Date | Y | nil | N | 收藏时间（用于排序） |
| `lastPlayedDate` | Date | Y | nil | Y | 最后播放时间 |
| `isDeleted` | Bool | N | false | Y | 软删除标记 |
| `deletedAt` | Date | Y | nil | N | 标记删除时间 |

**复合索引**：
- `idx_song_search`: (`title`, `artist`, `album`) — 搜索加速
- `idx_song_artist_album`: (`artist`, `album`) — 艺术家/专辑分组
- `idx_song_sort_name`: (`title`) — 按名称排序
- `idx_song_sort_date`: (`importDate`) — 按添加时间排序
- `idx_song_sort_size`: (`fileSize`) — 按文件大小排序
- `idx_song_hash`: (`fileHash`) — 去重查询
- `idx_song_active`: (`isDeleted`) — 过滤已删除记录

**Fetch Index**：
- `byArtist`: compoundIndexes = [artist ASC, album ASC, trackNumber ASC]
- `byAlbum`: compoundIndexes = [album ASC, trackNumber ASC]

#### 2.2.2 PlaylistEntity（歌单）

| 属性 | 类型 | 可选 | 默认值 | 索引 | 说明 |
|------|------|------|--------|------|------|
| `id` | UUID | N | auto | PK | 唯一标识 |
| `name` | String | N | "" | Y | 歌单名称（最多50字） |
| `desc_` | String | Y | nil | N | 描述（最多200字，避免关键字冲突） |

| `isSystem` | Bool | N | false | N | 系统歌单标记 |

| `updatedAt` | Date | N | now | N | 最后修改时间 |
| `sortOrder` | Int32 | N | 0 | Y | 排序权重 |

**Relationship**：
- `songs`: To-Many → `PlaylistSongEntity`（ordered，有序关系）

#### 2.2.3 PlaylistSongEntity（歌单-歌曲关联，中间表）

| 属性 | 类型 | 可选 | 默认值 | 索引 | 说明 |
|------|------|------|--------|------|------|
| `playlistId` | UUID | N | - | Y (复合) | 歌单ID |
| `songId` | UUID | N | - | Y (复合) | 歌曲ID |
| `sortOrder` | Int32 | N | 0 | N | 在歌单中的排序 |
| `addedAt` | Date | N | now | N | 加入歌单时间 |

**复合唯一约束**：(`playlistId`, `songId`) — 同一歌单不重复添加

**Relationship**：
- `playlist`: To-One → `PlaylistEntity` (Inverse: `songs`)
- `song`: To-One → `SongEntity`

> 使用中间表而非 CoreData 原生 Many-to-Many，原因：需要存储排序字段 `sortOrder` 和加入时间 `addedAt`。

#### 2.2.4 PlaylistGroupEntity（歌单分组）

| 属性 | 类型 | 可选 | 默认值 | 索引 | 说明 |

| `createdAt` | Date | N | now | N | 创建时间 |
| `sortOrder` | Int32 | N | 0 | Y | 排序权重 |

- `playlists`: To-Many → `PlaylistEntity` (Inverse: computed via groupId)

#### 2.2.5 PlayRecordEntity（播放记录）

| 属性 | 类型 | 可选 | 默认值 | 索引 | 说明 |
|------|------|------|--------|------|------|

| `playedAt` | Date | N | now | Y | 播放时间 |

> "最近播放"功能使用独立记录表，而非 Song.lastPlayedDate，支持按时间范围筛选和去重后排序。

#### 2.2.6 SearchHistoryEntity（搜索历史）

| 属性 | 类型 | 可选 | 默认值 | 索引 | 说明 |
|------|------|------|--------|------|------|

| `keyword` | String | N | "" | Y | 搜索关键词 |

**唯一约束**：(`keyword`, `tabContext`) — 同一 Tab 同一关键词去重

```swift
// MigrationManager.swift 伪代码
enum ModelVersion: String, CaseIterable {
    case v1_0 = "MusicPlayer_v1_0"

    var next: ModelVersion? {
        let all = Self.allCases
        guard let idx = all.firstIndex(of: self),
              idx + 1 < all.count else { return nil }
        return all[idx + 1]
    }
}
```

v1.0 为初始版本，后续版本升级使用 CoreData 轻量级迁移 (`NSMappingModel.inferredMappingModel`)。无法轻量级迁移时，编写显式 `NSEntityMigrationPolicy` 子类。

## 3. 核心服务层 API 设计

> 本应用为纯本地应用，无远程 API。以下为 Swift Service Layer 的协议定义（内部 API），供 ViewModel 调用。

### 3.1 音乐库服务（MusicLibraryService）

```swift
protocol MusicLibraryServiceProtocol {

    // MARK: - 查询

    /// 获取所有歌曲（支持排序）
    func fetchAllSongs(sortBy: SongSortOption, ascending: Bool) async -> [Song]

    /// 按艺术家分组获取
    func fetchSongsByArtist() async -> [ArtistGroup]

    /// 按专辑分组获取
    func fetchSongsByAlbum() async -> [AlbumGroup]

    /// 获取艺术家详情（含该艺术家所有专辑和歌曲）
    func fetchArtistDetail(artist: String) async -> ArtistDetail

    /// 获取专辑详情
    func fetchAlbumDetail(album: String, artist: String) async -> AlbumDetail

    /// 获取歌曲详情
    func fetchSongDetail(songId: UUID) async -> Song?

    /// 按格式筛选
    func fetchSongs(format: AudioFormat?) async -> [Song]

    /// 按收藏状态筛选
    func fetchSongs(favoriteOnly: Bool) async -> [Song]

    // MARK: - 删除

    /// 软删除歌曲（标记为待删除，30s 后物理清理）
    func softDeleteSong(_ songId: UUID) async throws -> UndoToken

    /// 批量软删除
    func softDeleteSongs(_ songIds: [UUID]) async throws -> UndoToken

    /// 撤销删除
    func undoDelete(token: UndoToken) async throws

    /// 执行物理删除（30s 到期后调用）
    func commitDelete(token: UndoToken) async throws

    /// 清理启动残留（tmp/pending_delete/）
    func cleanupPendingDeletes() async

    // MARK: - 收藏

    /// 切换收藏状态
    func toggleFavorite(songId: UUID) async throws

    /// 记录播放
    func recordPlay(songId: UUID) async throws
}
```

## 4. 关键流程设计

### 4.1 文件导入流程

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  UIDocument   │     │   Import     │     │   CoreData   │
│  Picker       │────►│  Service     │────►│   Stack      │
└──────────────┘     └──────┬───────┘     └──────────────┘
                           │
                    ┌──────┴───────┐
                    │              │
               ┌────▼────┐  ┌─────▼─────┐
               │ SHA256   │  │ Metadata  │
               │ Hash     │  │ Parser    │
               └────┬────┘  └─────┬─────┘
                    │              │
                    └──────┬───────┘
                           │
                    ┌──────▼───────┐
                    │  File Copy   │
                    │  to Sandbox  │
                    └──────────────┘
```

**详细步骤**：

1. 用户通过 `UIDocumentPickerViewController` 选择文件
2. 获取文件安全访问权限 (`url.startAccessingSecurityScopedResource()`)
3. 逐个文件处理：
   - a. 校验文件大小（≤500MB）
   - b. 校验文件扩展名（mp3/m4a/aac/alac/wav）
   - c. 计算 SHA256 哈希
   - d. 查询数据库是否已存在相同 hash 的 Song 记录
   - e. 若不存在：复制文件到 `Documents/Music/` 目录
   - f. 使用 `AudioMetadataParser` 解析元数据
   - g. 元数据缺失时按降级策略填充默认值
   - h. 创建 `SongEntity` 写入 CoreData
4. 释放安全访问权限 (`url.stopAccessingSecurityScopedResource()`)
5. 返回 `ImportResult` 统计

```
用户点击歌曲
```

```swift

    switch playMode {
    case .sequential:
        // 队列中当前位置之后，第一个 isPlayed == false 的项
```

## 7. 音频引擎设计

### 7.1 AudioEngine 封装

```swift
final class AudioEngine: ObservableObject {
    private var player: AVPlayer
    private var playerItem: AVPlayerItem?
    private var timeObserver: Any?

    @Published var state: PlaybackState = .idle
    @Published var currentTime: TimeInterval = 0
    @Published var duration: TimeInterval = 0
    @Published var isBuffering: Bool = false

    // 音频会话配置
    func configureAudioSession() throws {
        let session = AVAudioSession.sharedInstance()
        try session.setCategory(.playback, mode: .default)
        try session.setActive(true)
    }

    // 中断处理
    func handleInterruption(_ notification: Notification) {
        guard let info = notification.userInfo,
              let typeValue = info[AVAudioSessionInterruptionTypeKey] as? UInt,
              let type = AVAudioSession.InterruptionType(rawValue: typeValue)
        else { return }

        switch type {
        case .began:
            pause()
        case .ended:
            if let optionsValue = info[AVAudioSessionInterruptionOptionKey] as? UInt {
                let options = AVAudioSession.InterruptionOptions(rawValue: optionsValue)
                if options.contains(.shouldResume) {
                    play()
                }
            }
        @unknown default: break
        }
    }
}

enum PlaybackState {
    case idle
    case loading
    case playing
    case paused
    case error(Error)
}
```

### 8.4 ThemeManager（全局主题管理器）

```swift
@Observable
final class ThemeManager {

    // 持久化存储
    @ObservationIgnored
    @AppStorage("color_mode") private var storedColorMode: Int = 0
    @ObservationIgnored
    @AppStorage("theme_style") private var storedThemeStyle: String = "morandi_green"

    // 当前状态
    var colorMode: ColorMode {
        get { ColorMode(rawValue: storedColorMode) ?? .system }
        set { storedColorMode = newValue.rawValue }
    }

    var currentStyle: ThemeStyle {
        get {
            ThemeStyle.allStyles.first { $0.id == storedThemeStyle }
                ?? .morandiGreen
        }
        set { storedThemeStyle = newValue.id }
    }

    // 根据当前 colorScheme 解析出实际色板
    func resolvedPalette(for colorScheme: ColorScheme) -> ThemeColorPalette {
        colorScheme == .dark ? currentStyle.darkPalette : currentStyle.lightPalette
    }

    // 提供给 .preferredColorScheme() 使用
    var resolvedColorScheme: ColorScheme? {
        colorMode.colorScheme
    }
}
```

### 10.1 列表性能

| 策略 | 实现方式 | 适用场景 |

### 10.2 内存控制

| 限制项 | 阈值 | 策略 |
|--------|------|------|
| 图片缓存 (NSCache) | 50MB | LRU 淘汰 |
| 图片磁盘缓存 | 100~300MB | 动态上限，LRU |
| 音频 buffer | 当前1首 + 下一首元数据 | AVPlayer 自动管理 |

| 播放队列 | 999 首 | 硬上限 |
| 搜索历史 | 10 条/Tab | 超出删除最旧 |

### 10.3 启动优化

1. CoreData 栈异步初始化（不阻塞 UI）
2. 首屏仅加载必要数据（当前 Tab 前 50 首）
3. 图片缓存预热延迟到空闲时
4. 清理 `tmp/pending_delete/` 在后台 Task 执行
