### v4.3 主要变更（基于v4.2问题分析文档采纳全部16项）

**数据模型重构（问题1/2/4/11）**：
1. **Song模型瘦身**：`coverImage: Data?` 改为 `coverImagePath: String?`，`lyrics: String?` 移除（按需延迟加载），避免列表场景内存暴涨
2. **补充核心模型**：新增 Playlist、PlaylistGroup、Metadata 完整结构定义
3. **PlayQueue重构**：引入 `QueueItem` 包装结构，携带 `isPlayed` 和 `addedAt` 状态
4. **PlayQueueConfig语义修正**：明确为编译期常量（`static let`），与用户可配置项分离

**逻辑冲突修复（问题3/5/6）**：
5. **文件删除与hash去重统一**：删除Song记录后，通过引用计数判断是否删除物理文件，避免共享文件被误删
6. **播放模式与队列交互**：按顺序/单曲循环/列表循环/随机四种模式，分别明确下一首选择、末尾行为、已播放标记重置规则
7. **删除撤销实现路径**：采用"软删除+延迟清理"方案（移至temp目录，30秒后清理）

**边界条件补全（问题7/8/9/10）**：
8. **WiFi传输断连保护**：旧连接正在传输时，新连接需手机端确认才断开；中断文件自动清理
9. **文件导入大小限制统一**：文件应用导入与WiFi传输统一为单文件500MB上限
10. **"我喜欢"歌单完整定义**：补充显示位置、不可删除/重命名、排序规则、分组归属、封面生成规则
11. **元数据解析降级策略**：标题用文件名、艺术家/专辑显示"未知"、时长从音频流获取

**设计优化（问题12/13/14/15/16）**：
12. **歌单命名规则放宽**：移除文件系统特殊字符限制（歌单存储在数据库），导出时做字符转义
13. **图片缓存上限调整**：默认100MB，上限300MB，设备可用存储<1GB时自动降级
14. **FLAC筛选条件化显示**：FLAC解码器未集成时，筛选项标注"暂不支持播放"
15. **通知权限前置检查**：用户开启定时播放时立即检查通知权限，未授权则引导开启
16. **歌曲详情页设计**：补充完整页面布局、展示字段、与P3元数据编辑的关系

## 1. 产品概述

### 1.1 产品定位
一款专注于iOS平台的本地音乐播放器，致力于为用户提供简洁、优雅、功能完善的本地音乐管理和播放体验。

**核心价值**：
- 🎵 完全本地播放，无需网络
- 📁 灵活的音乐组织方式
- 🎨 简洁优雅的用户界面
- ⚡️ 快速响应，流畅体验
- 🔒 保护用户隐私，数据不上传

### 1.2 产品愿景
成为iOS平台最好用的本地音乐播放器，让用户轻松管理和享受自己的音乐收藏。

### 1.3 目标用户
- **主要用户群体**：拥有大量本地音乐文件的用户
- **年龄范围**：18-45岁
- **使用场景**：日常通勤、运动、工作、睡前等需要本地音乐播放的场景
- **核心需求**：
  - 需要方便地导入和管理本地音乐文件
  - 需要多种播放模式满足不同场景
  - 需要良好的音乐组织和分类功能
  - 需要简洁美观的操作界面

### 1.4 技术限制说明
**重要**：本产品受iOS平台限制，以下功能无法实现或受限实现：
- ❌ 无法扫描整个手机文件系统（iOS沙盒限制）
- ❌ 无法修改从Apple Music库导入的文件（只读限制）
- ⚠️ 只能访问应用沙盒和用户授权的目录
- ⚠️ 部分功能需要用户明确授权
- ⚠️ 后台任务受系统限制，定时器可能不精确

### 2.2 数据模型

```swift
// ─── 音乐数据模型（v4.4完善） ───

struct Song {
    let id: String                    // 唯一标识（UUID）
    var title: String                 // 歌曲名称
    var artist: String                // 艺术家
    var album: String                 // 专辑
    let duration: TimeInterval        // 时长（秒）
    let fileHash: String              // 文件内容hash（用于去重，多Song可共享同一物理文件）
    let fileURL: URL                  // 文件路径（相对Documents/Music目录）
    var coverImagePath: String?       // 封面图片文件路径（相对Caches目录，按需加载）
    var lyricsPath: String?           // v4.4新增：外置歌词文件路径（相对Documents/Lyrics目录）
    var lyricsSource: LyricsSource?   // v4.4新增：歌词来源（nil表示未加载歌词）
    let metadata: Metadata            // 音频元数据
    let importDate: Date              // 导入时间
    let fileSize: Int64               // 文件大小（字节）
    var playCount: Int                // 播放次数
    var isFavorite: Bool              // 是否收藏
    var lastPlayedDate: Date?         // 最后播放时间
}

// ─── 歌词来源枚举（v4.4新增） ───

enum LyricsSource {
    case embedded    // 内嵌在ID3标签（LYRIC/USLT）
    case file        // 外置.lrc文件（与音频文件同名）
    case manual      // 用户手动添加（存储在Documents/Lyrics/目录）
}

// 说明：
// - coverImage 不再内联存储 Data，改为文件路径引用，列表加载时不读取图片二进制
// - lyricsPath + lyricsSource：管理延迟加载的歌词，支持内嵌/外置文件/手动添加三种来源
// - fileHash 用于导入去重，多个 Song 记录可能指向同一物理文件

// ─── 音频元数据 ───

struct Metadata {
    let format: String                // 音频格式（mp3/m4a/aac/alac/wav/flac等）
    let sampleRate: Int?              // 采样率（Hz），如 44100
    let bitRate: Int?                 // 比特率（kbps），如 320
    let channels: Int?                // 声道数，如 2
    let encoder: String?              // 编码器信息
    let trackNumber: Int?             // 音轨编号
    let year: Int?                    // 发行年份
    let genre: String?                // 流派
}

// ─── 歌单模型（v4.4完善） ───

struct Playlist {
    let id: String                    // 唯一标识（UUID）
    var name: String                  // 歌单名称（最多50字）
    var description: String?          // 歌单描述（最多200字）
    var customCoverPath: String?      // 用户自定义封面路径（优先级高于autoCoverPath）
    var autoCoverPath: String?        // v4.4新增：自动生成的拼接封面路径（相对Caches/PlaylistCovers/目录）
    var autoCoverHash: String?        // v4.4新增：前4首歌曲的hash（用于判断是否需要重新生成）
    var songIds: [String]             // 包含的歌曲ID列表（有序）
    var groupId: String?              // 所属歌单分组ID（nil表示未分组）
    let isSystem: Bool                // 是否为系统歌单（"我喜欢"、"最近播放"）
    let createdAt: Date               // 创建时间
    var updatedAt: Date               // 最后修改时间
    var sortOrder: Int                // 在歌单列表中的排序位置
}

// 歌单封面逻辑（v4.4明确）：
// 1. 显示封面时：如果customCoverPath不为nil，使用自定义封面；否则使用autoCoverPath
// 2. autoCoverPath为nil或autoCoverHash与当前前4首不匹配时，重新生成2x2拼接封面
// 3. 生成后保存到Caches/PlaylistCovers/目录，路径存入autoCoverPath
// 4. 清理缓存时，可以删除autoCoverPath，但不删除customCoverPath

// ─── 歌单分组模型（v4.3新增） ───

struct PlaylistGroup {
    let id: String                    // 唯一标识（UUID）
    var name: String                  // 分组名称
    let createdAt: Date               // 创建时间
    var sortOrder: Int                // 在分组列表中的排序位置
}

// ─── 用户播放偏好 ───

struct UserPreferences {
    var songDislikes: [String: Int]   // songId -> dislikeCount
    // dislikeCount上限：5次
    // 提供"重置不喜欢"功能
}

// ─── 播放队列（v4.3重构） ───

struct QueueItem {
    let song: Song                    // 歌曲引用
    var isPlayed: Bool                // 是否已播放
    let addedAt: Date                 // 加入队列时间
}
typealias PlayQueue = [QueueItem]

// 队列常量配置（编译期固定值，非用户可配置）
enum PlayQueueConstants {
    static let maxCount = 999         // 最大容量
    static let persist = false        // 不持久化（应用重启后清空）
    static let removePlayed = false   // 已播放歌曲保留在队列中
    static let markAsPlayed = true    // 标记为已播放（用于UI区分）
}
```

2. **单曲循环**：循环播放当前歌曲
3. **列表循环**：播放完列表后重新开始

   - 提供"不喜欢"按钮，降低当前歌曲的随机权重

#### 3.2.3 播放队列 (P1) - 明确逻辑

**v4.0明确定义**：
- 播放队列是**临时的**播放上下文记录
- 最大容量：999首歌曲
- **不持久化**到磁盘（应用重启后清空）
- **已播放的歌曲保留在队列中**（标记为已播放）

**队列管理功能** (P2)：
- 查看当前播放队列
- 拖拽调整播放顺序
- 移除队列中的歌曲
- 清空队列
- 添加歌曲到队列
- 保存队列为新歌单（创建副本）

**队列行为**：
- 当前播放的歌曲始终在队列中
- 已播放的歌曲保留在队列中（QueueItem.isPlayed = true，用于UI区分）
- 应用重启后队列清空

**播放模式与队列交互规则**（v4.3新增）：

| 播放模式 | 下一首选择逻辑 | 到达末尾后的行为 | 已播放标记(isPlayed)重置时机 |
|---------|-------------|---------------|------------------------|
| **顺序播放** | 队列中当前位置的下一首（跳过已播放项） | 所有未播放项播完后**停止播放** | 不自动重置 |
| **单曲循环** | 重复播放当前歌曲 | 不涉及末尾概念 | 不标记为已播放（持续循环） |
| **列表循环** | 队列中当前位置的下一首（跳过已播放项） | 所有项播完后，**重置全部isPlayed为false**，从第一首重新开始 | 每轮循环结束时全部重置 |
| **随机播放** | 从队列中isPlayed=false的项中按权重随机选择 | 所有项都已播放后**停止播放** | 不自动重置 |

- 用户手动切换播放模式时，不重置已播放标记
- 用户手动点击队列中已播放的歌曲时，正常播放该歌曲（不受isPlayed限制）

**队列满（999首）处理策略（v4.1新增）**：
- 当队列达到999首且用户继续添加时，弹窗提示：`播放队列已满（999首）`
- 提供选项：
  1. `清空后添加`（清空当前队列并添加新歌曲）
  2. `仅播放不入队`（立即播放，不改变队列）
  3. `取消`
- 默认高亮 `仅播放不入队`（降低误操作风险）

#### 3.2.4 定时播放 (P2) - 可实现版（v4.1修订）

**交互逻辑**：

- 定时结束前1分钟可选发送本地通知（需授权）
- 时间到后执行：暂停播放 + 轻触觉反馈（前台可用）
- 提供重置选项（恢复默认）

**通知权限检查时机**（v4.3新增）：
- 用户**开启定时播放时**立即检查通知权限状态

**验收标准（v4.2补充）**：
- **前台**：按目标时间准时暂停
- **后台/锁屏**：尽力保证，受iOS系统调度影响，允许±60秒误差窗口
- **兜底机制**：应用恢复活跃时立即校验"当前时间 >= 目标结束时间"，若已超时则补执行暂停
- **用户提示**：设置定时器时显示说明文案"后台定时暂停受系统调度影响，可能存在少许延迟"

**用户操作流程**：
```
点击定时器图标 → 选择时间 → 开始倒计时
→ 应用内显示剩余时间 → 最后一分钟可选通知
→ 震动反馈 → 停止播放
```

**技术方案（v4.1新增）**：
- 使用“目标结束时间戳”而非长期依赖前台Timer
- 前台：每秒刷新倒计时UI
- 后台/锁屏：应用恢复活跃时以`当前时间 >= 目标结束时间`校验并执行暂停
- 文案提示：`离开应用后，倒计时展示可能不实时，结束动作仍按目标时间校验执行`

### 3.8 性能优化 (P1)

#### 3.8.1 图片缓存策略 (P1)

**v4.3变更**：缓存上限下调，匹配本地封面实际场景

**缓存范围说明**：

**缓存配置**（v4.3调整）：
- 缓存大小上限：`min(可用存储空间的5%, 300MB)`

- 下限：50MB

**缓存清理**：
- 自动清理：达到动态上限时，清理最久未使用的20%
- 手动清理：设置页提供"清理缓存"按钮
- 清理时显示已用/可用空间
- 设备存储紧张时（< 500MB可用）：弹出建议清理缓存的提示

**大文件处理**：
- 大图片自动压缩（宽度和高度不超过1024px）
- 生成缩略图（100x100px、200x200px）
- 使用INProgressiveLoading（渐进式加载）

#### 3.8.2 大文件播放 (P1)

**v4.0变更**：澄清技术方案

**文件大小限制**：
- 单个文件不超过500MB
- 超过限制时提示用户

**内存映射播放**（v4.0澄清）：
- 对于大文件（>50MB），使用**内存映射播放**
- AVPlayer默认支持，不需要完整加载到内存
- 不需要"边下载边播放"（这是流媒体播放的术语）
- 当前歌曲播放进度达到80%时，预加载下一首元数据（不是整个文件）
- 确保切换歌曲时无延迟

**内存管理**：
- 同时播放的歌曲数量：1首
- 预加载下一首：优先仅元数据；若设备空闲且电量充足，可额外预热前500KB音频数据
- 播放完立即释放资源

### 3.12 歌词显示 (P2) - v4.4完善加载机制

**v4.4补充：歌词延迟加载流程**

**第三方库选型（v4.4明确）**：
- **v1.0使用ID3TagEditor库**（MIT许可证）读取ID3标签
- 支持格式：MP3、M4A等主流音频格式的元数据解析
- GitHub：https://github.com/chrisaiv/ID3TagEditor

**歌词加载优先级**（按以下顺序尝试）：
1. **内嵌歌词**（ID3v2 tags）：
   - ID3v2 LYRIC（纯文本）
   - ID3v2 USLT（同步歌词，带时间轴）
   - 使用ID3TagEditor库解析
   - 加载成功后设置 `lyricsSource = .embedded`

2. **外置LRC文件**：
   - 与音乐文件同名的.lrc文件（位于Documents/Lyrics/目录）
   - 文件命名规则：`歌曲名.lrc` 或 `歌曲名-艺术家.lrc`
   - 支持同步歌词（时间轴）
   - 加载成功后设置 `lyricsSource = .file`，`lyricsPath = "Lyrics/xxx.lrc"`

3. **用户手动添加歌词**（P3）：
   - 用户在歌曲详情页手动输入LRC格式歌词
   - 保存到Documents/Lyrics/目录（文件名：`{songId}.lrc`）
   - 加载成功后设置 `lyricsSource = .manual`，`lyricsPath = "Lyrics/{songId}.lrc"`

**歌词显示页加载流程**（v4.4明确）：
```
打开歌词显示页
→ 检查Song.lyricsSource
  → 如果为.embedded：从ID3标签解析歌词（异步加载，不阻塞UI）
  → 如果为.file或.manual：从lyricsPath加载文件内容
  → 如果为nil：按优先级依次尝试加载（内嵌 → 外置文件）
→ 显示歌词（解析失败或无歌词时显示"暂无歌词"）
→ 用户手动添加/编辑歌词（P3）：保存到Documents/Lyrics/，更新lyricsPath和lyricsSource
```

**歌词存储规范**（v4.4明确）：
- 内嵌歌词：不存储，每次从ID3标签读取
- 外置LRC文件：存储在Documents/Lyrics/目录
- 用户手动添加：存储在Documents/Lyrics/目录，文件名使用songId
- 编辑歌词（P3）：只修改lyricsPath指向的文件，不修改原始音频文件

**不支持**（v2.0功能）：
- 在线歌词匹配
- 歌词翻译
