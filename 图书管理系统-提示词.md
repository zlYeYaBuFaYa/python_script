## 图书管理系统-提示词

- 我想实现一个 web 端的图书管理系统，请帮我生成一个关于需求，功能的提示词

#### ds:

```
1. 项目概述
项目名称： Web端图书管理系统

项目目标： 开发一个基于B/S（浏览器/服务器）架构的图书管理系统，旨在实现对图书馆或资料室藏书的数字化、自动化管理。系统需提升图书检索、借阅、归还等流程的效率，并为管理员提供便捷的图书入库、分类和数据统计功能。

目标用户：

    普通读者/市民： 查询图书、查看个人借阅信息。

    图书馆管理员： 负责所有图书和借阅流程的管理。

    核心价值： 简化图书管理流程，减少人工错误，提高图书流通率和信息透明度。

2. 用户角色与核心用例
    角色一：读者

        用例1： 注册与登录个人账户。

        用例2： 按书名、作者、ISBN、分类等条件检索图书。

        用例3： 查看图书详情（书名、作者、出版社、简介、馆藏数量、可借数量）。

        用例4： 查看个人借阅记录（当前借阅、借阅历史）。

        用例5： 续借图书（在规定的期限内）。

        用例6： 修改个人基本信息（如密码、联系方式）。

    角色二：系统管理员
        用例1： 管理员登录与权限验证。

        用例2： 对图书进行增、删、改、查操作。

        增： 支持单本录入或通过Excel批量导入图书信息（ISBN、书名、作者、出版社、出版日期、分类、价格、总数量等）。

        改： 更新图书信息，如图书损坏、丢失后修改馆藏数量。

        用例3： 管理图书分类，可新增、编辑、删除分类。

        用例4： 管理用户账户，可启用、禁用、重置用户密码。

        用例5： 处理借书、还书、续借操作。

        用例6： 查看所有借阅记录，并能按用户、图书、时间进行筛选。

        用例7： 管理借阅规则，如设置最大借书数量、借阅期限、逾期罚款规则。

        用例8： 查看系统数据统计仪表盘，如图书借阅排行榜、新增图书趋势、活跃用户统计等。

3. 核心功能模块详述
    模块A：前台读者门户
        用户认证模块

            用户注册（需验证邮箱或手机号）。

            用户登录/登出。

            密码找回功能。
            用户注销

        图书检索与浏览模块

            搜索框： 支持关键词模糊搜索（书名、作者、出版社）。

            高级搜索： 提供按分类、ISBN、出版年份等组合筛选。

            分类导航： 以树形结构或标签页展示图书分类，点击分类可浏览该分类下所有图书。

            图书列表页： 分页展示图书，显示封面（缩略图）、书名、作者、可借状态。

            图书详情页： 展示完整信息，包括馆藏总数和可借数量。

        个人中心模块

            我的借阅： 表格形式展示“当前借阅”（含应还日期）和“借阅历史”。

            续借功能： 在“当前借阅”列表中，每本书提供“续借”按钮，每本书仅可续借一次。

            个人信息： 展示和编辑姓名、邮箱等非敏感信息。

    模块B：后台管理面板

        仪表盘模块

            显示关键数据卡片：图书总数、用户总数、当日借阅量、逾期未还数量。

            图表展示：近30天借阅趋势图、热门借阅图书TOP 10。

        图书管理模块

            图书列表： 提供增、删、改、查、批量导入/导出功能。

            图书入库： 表单包含：ISBN（支持扫码枪录入）、书名、作者、出版社、出版日期、分类（下拉选择）、定价、入库数量。

        分类管理： 可对图书分类进行增删改查。

用户管理模块

            查看所有注册用户列表。

            支持按姓名、手机号，邮箱搜索用户。

            可手动添加用户（如为不熟悉系统的管理员添加账户）。

            可禁用/启用账户、重置用户密码。

        借阅管理模块

            借书操作： 输入用户ID/姓名和图书ISBN/书名，系统自动校验用户资格和图书可借状态后完成借阅。

            还书操作： 输入图书ISBN或扫描条形码，系统自动计算是否逾期，并显示逾期费用（如有）。

            借阅记录： 查看所有历史借阅记录，支持多条件筛选和导出为Excel。

        系统设置模块

        借阅规则设置： 可配置“最大借书数量”、“借阅期限（天）”、“续借期限（天）”、“逾期每日罚款金额”。

4. 非功能性需求
    性能： 页面加载时间应小于3秒，支持至少100个用户并发操作。

    安全性：

        用户密码需加密存储（如bcrypt）。

        防止SQL注入和XSS攻击。

        管理员操作需进行会话管理和权限校验。

        普通用户不能访问后台管理页面。

    可用性：

        界面设计简洁、直观，符合常见Web操作习惯。

        提供清晰的操作反馈（成功、失败、错误提示）。

        可靠性： 系统应保持高可用性，数据需定期备份。

5. 技术栈建议（可选）
    前端： Vue.js  + Element UI

    后端： gin(golang)

    数据库： MySQL / PostgreSQL

    其他： RESTful API 设计

6. 交付物要求
    完整的前后端源代码，并带有清晰的代码注释。

    数据库设计文档（ER图）和SQL脚本。

    部署文档，说明如何配置环境和运行项目。

    用户使用手册（可选）。
```
